# XmlConverter.Web
Проект реализован строго по требованиям задания. Основная логика - приём XML, валидация, конвертация и работа с данными - выполнена без базы данных и ORM: все данные хранятся в памяти, что не перечит условиям ТЗ

Если такой подход считается минусом, могу быстро переработать решение на устойчивое хранение (например, SQLite + ORM), чтобы документы сохранялись между перезапусками

Также не была добавлены поддержка Data2 к изменению - по условиям ТЗ требования только для Data1

## Запуск
1. `dotnet run`
2. Откройте URL из консоли (например, `https://localhost:5141/`).
3. Достаточно перейти на:
   - `/` - загрузка XML
   - `/employees` - просмотр и работа с данными (если файл уже загружен в память)

## API
- `POST /converter/upload` - загрузка XML (body: raw XML)
- `GET /converter/converted` - получить конвертированный XML
- `GET /converter/data` - получить исходный XML
- `POST /converter/append_data` - добавить item
  - query:
    - `name` - `string`
    - `surname` - `string`
    - `amount` - `decimal`
    - `month` - `string`
